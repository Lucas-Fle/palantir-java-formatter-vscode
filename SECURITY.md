# Politique de sécurité

## Versions supportées

| Version | Support de sécurité |
| --- | --- |
| Dernière version publiée | Oui |
| Versions antérieures | Non |

Les correctifs de sécurité sont publiés uniquement sur la dernière version de
l’extension. Les utilisateurs doivent donc mettre à jour avant de signaler un
problème déjà corrigé.

## Signaler une vulnérabilité

Ne pas ouvrir d’issue publique pour une vulnérabilité potentielle. Utiliser le
bouton **Report a vulnerability** dans l’onglet **Security** du dépôt GitHub
afin de créer un rapport privé.

Le rapport doit contenir, dans la mesure du possible :

- la version de l’extension, de VS Code et du JDK ;
- le système d’exploitation ;
- les étapes minimales de reproduction ;
- l’impact observé ou potentiel ;
- toute proposition de correction, si elle est connue.

Ne pas joindre de code source confidentiel, de jeton ou de donnée personnelle.
